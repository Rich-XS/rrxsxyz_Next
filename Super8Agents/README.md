## 🎯 系统简介

Super8Agents 是一个由 8 位领域专家组成的 AI 辩论决策系统，专注于 **RRXS.XYZ 项目的商业化落地和技术实施**。

**核心特性**:
- 🧠 **8 位专家**：商业/产品/技术/市场/设计/增长/数据/执行
- 🔄 **PDCA 迭代**：每个话题 20 轮深度辩论（Plan → Do → Check → Act）
- 📊 **数据驱动**：所有决策基于真实数据和行业案例
- ⚡ **可执行性**：每轮产出具体行动计划（含时间/成本/资源）
- 🎭 **领袖协调**：Claude Code 作为领袖，整合专家建议，确保决策质量

## 🔄 PDCA 辩论流程

每个话题进行 **20 轮 PDCA 迭代**，每轮包含 4 个阶段：

### Plan（计划阶段）
- **目标**: 明确本轮要解决的核心问题
- **参与**: 全体 8 位专家轮流发言，提出各自视角的规划建议
- **产出**: 初步行动方案（包含目标/方法/指标）

### Do（执行阶段）
- **目标**: 细化执行细节，识别资源需求和潜在风险
- **参与**: 执行官（钱敏）牵头，其他专家补充
- **产出**: 详细执行计划（包含时间节点/责任人/资源清单）

### Check（检查阶段）
- **目标**: 评估方案可行性，识别盲点和改进空间
- **参与**: 数据分析师（孙涛）牵头，其他专家质疑和挑战
- **产出**: 风险清单和改进建议

### Act（改进阶段）
- **目标**: 整合反馈，优化方案，确定最终行动计划
- **参与**: 领袖（Claude Code）总结，全体确认
- **产出**: 本轮最终决策和下一轮输入

## 🎯 辩论质量保证

### 5 大质量要求

1. **数据真实性**: 所有数据和案例必须真实可验证（12个月内原则）
2. **建议具体性**: 每位专家必须提供 3 个可执行建议（含时间/成本/资源）
3. **角色一致性**: 严格遵守角色定位，不得混淆
4. **递进要求**: 后续轮次必须在前轮基础上深化，避免重复
5. **字数控制**: 每位专家单次发言 300-500 字（确保深度但不冗长）

### 产出标准

- ✅ **可执行性**: 每个建议必须有具体时间节点和责任人
- ✅ **数据支撑**: 每个结论必须有数据或案例支撑
- ✅ **风险识别**: 每个方案必须识别潜在风险和应对措施
- ✅ **资源清单**: 每个计划必须明确资源需求（人力/预算/工具）
