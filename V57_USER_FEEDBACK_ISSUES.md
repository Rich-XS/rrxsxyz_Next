## 🔴 **Critical Issues** (需立即修复)

### Issue #1: 轮数计算BUG
- **现象**: 用户选择'3轮'，但系统生成了**4轮**
- **表现**:
  1. 第1轮 ✅
  2. 第2轮 ✅
  3. 第3轮 ✅
  4. **第4轮异常** ❌ (不应该存在)
  5. 领袖直接第4轮总结
  6. 全场总结
- **根本原因**: debateEngine.js 轮数计算逻辑
- **影响**: 用户期望3轮，实际4轮
- **优先级**: P0

### Issue #2: FINAL总结显示不一致
- **现象**:
  - 第1轮结束显示 ✅ `【总结】` 标签（图1）
  - 第2轮结束 ❌ 没有 `【总结】` 标签（图2）
- **代码位置**: index.html L403（typeLabel定义）或debateEngine.js（summary类型判断）
- **优先级**: P1

### Issue #3: 内容逻辑错误
- **现象**: "邀请增长黑客专家分析变现漏斗模型..."
- **问题**: 应该邀请**当前风暴辩论中的专家角色**，而不是其他专家
- **根本原因**: promptTemplates.js 中的邀请逻辑
- **优先级**: P1

## 🟡 **Performance Issues** (用户体验优化)

### Issue #4: 领袖开场速度慢
- **现象**: 第一轮领袖开场有明显延迟
- **影响**: 用户感知体验不流畅
- **可能原因**:
  - AI生成延迟（网络/模型）
  - 流式传输初始延迟
- **优先级**: P2

### Issue #5: 第一性原则开场速度慢
- **现象**: 第一个角色（第一性原则）发言启动较慢
- **可能原因**: 模块初始化或首次TTS/流式输出初始化
- **优先级**: P2

## 🟠 **Feature Gaps** (功能缺陷)

### Issue #6: 性别语音选择未生效
- **现象**:
  - Console看到性别识别 ✅ (9男+7女)
  - 但**听不到男声**（例如Elon第一性原则应该是男声）
- **根本原因**:
  - VoiceModule.selectVoice() 可能没有真正应用性别对应的语音
  - 或者没有预加载男声
- **代码位置**: voice.js L63-652
- **优先级**: P1

### Issue #7: 角色显示格式不统一
- **现象**:
  - 领袖显示为 "Victoria"
  - 其他显示为 "行业专家" 等
- **建议**: 改为 "**角色+名字**" 格式
  - 例: "领袖(Victoria)" / "第一性(Elon)" / "行业专家(Jane)"
- **代码位置**: index.html L587-588（speech-header）
- **优先级**: P2

### Issue #8: 发言内容重复
- **现象**: 第2轮开始，每位专家发言都以"作为XXX..."开头
- **问题**: 显得冗余、不够自然
- **建议**: promptTemplates.js 中针对非首轮，减少"作为XXX"的引入
- **优先级**: P2

### Issue #9: 委托人补充内容未显示
- **现象**:
  - 第一轮结束有bonus环节，用户补充了"GOOD"
  - 领袖在最终总结中结合了补充 ✅
  - **但补充内容本身没有在UI中显示出来**
- **建议**: 在领袖补充后/总结前，把委托人补充文本也显示出来
- **代码位置**: index.html delegatePrompt渲染逻辑
- **优先级**: P2

## 🟢 **UI/UX Improvements** (下一版本优化)

### Issue #10: 首轮标签颜色优化
- **现象**: ✅ 蓝#007AFF + 橙#FF9500 双层关键词系统很棒！
- **建议**:
  - 橙色标题字体**可再大一些**
  - 考虑**多颜色方案**（橙/紫色/绿色）选择最终效果
- **优先级**: P3 (Enhancement)

### Issue #11: 底部按钮体验
- **现象**: 多专家连续发言时，委托人需要滚动查看内容，但按钮（打断/暂停/补充/确认）在页面底部
- **用户需求**: 考虑**悬浮窗形式**（顶部Banner区域或底部底边栏固定）
- **优先级**: P3 (V58+)
- **建议**: 评估可行性
